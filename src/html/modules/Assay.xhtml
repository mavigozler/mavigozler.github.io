Cut or copy and then paste the x and y coordinate values in the appropriate boxes. This can usually be done by selecting values in another application, such as in a database table or report, or in a spreadsheet like Excel.

The values can be separated either by commas or whitespace ( whitespace is either one or more space characters or one or more tab characters or one or more linefeeds and/or carriage return characters). Do not mix the separator character for values: for example, do not start using a comma and then switch to a space or linefeed or carriage return.

Value must be real numbers: they can be integers (whole numbers), floating point, or in scientific notation. The proper format for scientific notation is <mantissa>e[−]<exponent>. Some examples: 2.97 × 10−5 should be written 2.97e−5. −9.600×1012 is −9.600e12. Note that there are no spaces. Positive values of either the mantissa or exponent do not require plus (‘+’) signs, but values less than zero always require a minus sign or hyphen. Any entry which contains unrecognized characters within a string of non-whitespace characters will be considered a non-entry (and converted to a single space for the purpose of evaluation).

There must be a one-to-one correspondence between values. If there is a mismatch in the count of values between the x and y entries, no evaluation will be done and an error message will be reported indicating a correction should be done.
